## 1. 游戏概述

### 1.1 游戏定位
EarthLord是一款将现实世界GPS定位与末日生存题材结合的创新型手机游戏。玩家通过现实中的步行来圈占领地、探索废墟、收集资源,在虚拟的末日世界中重建文明。

### 1.2 核心卖点
1. **Walk to Earn**: 走路即游戏,鼓励现实世界的户外活动
2. **真实地理融合**: 将真实世界的地标转化为游戏内的资源点
3. **领地所有权**: 玩家可以永久拥有现实世界对应的虚拟领地
4. **以物易物经济**: 去中心化的玩家交易系统
5. **希望朋克美学**: 独特的废土新生视觉风格

### 1.3 目标玩家
- **核心玩家**: 20-35岁,喜欢探索和收集,有户外活动习惯
- **休闲玩家**: 18-45岁,通勤族,希望步行有额外收益
- **社交玩家**: 喜欢与他人互动,建立联盟,协作游戏
- **竞技玩家**: 追求排行榜排名,展示成就

### 1.4 游戏目标
- 短期目标: 圈占第一块领地,完成第一次探索
- 中期目标: 建立资源生产循环,扩展领地,提升等级
- 长期目标: 成为区域领主,建立联盟,主宰排行榜

## 3. 核心玩法机制

### 3.1 领地占领机制

#### 3.1.1 圈地系统
**玩法描述**:
玩家通过现实世界的步行,用GPS轨迹在地图上画出闭合多边形,从而占领领地。

**操作流程**:
1. 打开游戏,点击"开始圈地"
2. 开始步行,GPS实时记录轨迹
3. 轨迹显示为蓝色线条
4. 走回起点附近(20米内),轨迹自动闭合
5. 系统验证合法性
6. 验证通过,领地变为绿色多边形

**合法性验证**:
- ✅ 面积在500-100,000m²之间
- ✅ 轨迹点间隔合理(5-10米)
- ✅ 形成有效闭合多边形
- ✅ 不与其他玩家领地重叠
- ❌ GPS漂移检测(速度异常)
- ❌ 作弊检测(传送、飞行)

**奖励机制**:
- 获得领地所有权
- 解锁建造权限
- 获得初始资源奖励
- 成就进度增加

#### 3.1.2 领地类型
| 领地类型 | 面积要求 | 特殊效果 |
|---------|---------|---------|
| 小型营地 | 500-2,000m² | 基础建造 |
| 中型基地 | 2,000-10,000m² | 生产加成+10% |
| 大型据点 | 10,000-50,000m² | 生产加成+20%,防御+10% |
| 超级城邦 | 50,000-100,000m² | 全属性+30%,可设立税收 |

#### 3.1.3 领地管理
- **命名**: 自定义领地名称
- **标记**: 添加颜色标签和图标
- **权限设置**:
  - 禁止交易/允许交易
  - 私人领地/公共领地
  - 联盟共享/个人独占
- **维护机制**:
  - 30天内必须有活动
  - 90天不活跃将被回收
  - 可以设置领地管理员(未来版本)

### 3.2 探索与搜刮机制

#### 3.2.1 POI发现
**触发条件**:
- 玩家进入POI 100米范围内
- 该POI未在24小时内被该玩家搜刮过
- 玩家探索次数未达到当日上限(免费:10次/天,会员:无限)

**发现流程**:
1. 系统检测到玩家进入范围
2. 地图上POI图标高亮
3. 弹出"发现兴趣点"提示
4. 玩家点击"搜刮"开始探索

#### 3.2.2 搜刮机制
**基础规则**:
```
获得资源数量 = 基础产出 × POI稀有度系数 × 探索距离加成 × 会员加成
```

**计算公式**:
- 基础产出: 10-30个随机物品
- POI稀有度:
  - 普通(超市): ×1.0
  - 稀有(医院): ×1.5
  - 史诗(银行): ×2.0
- 探索距离加成:
  - <1km: ×1.0
  - 1-5km: ×1.2
  - 5-10km: ×1.5
  - >10km: ×2.0
- 会员加成:
  - 免费: ×1.0
  - 月卡: ×1.5
  - 通行证: ×2.0

**产出物品示例**:
搜刮"废弃超市"可能获得:
- 罐头食品 ×24 (恢复饱食度40)
- 饮用水 ×12 (恢复水分30)
- 布料 ×7 (建造材料)
- 金属碎片 ×3 (建造材料)

#### 3.2.3 探索成就
每次探索结束后显示:
- 探索距离: X.XX km
- 探索时长: XX min
- 获得物品清单
- 解锁成就(如果有)
- 经验值增加

#### 3.4.2 建造流程
1. 进入建造界面
2. 选择建筑类型
3. 查看建造条件:
   - 等级要求 ✓/✗
   - 资源是否足够 ✓/✗
   - 领地是否有空间 ✓/✗
4. 点击"开始建造"
5. 拖动图标选择位置
6. 确认建造
7. 资源扣除
8. 建筑进入"建造中"状态
9. 等待完成(即时完成或需要时间)
10. 建筑变为"待激活"状态

#### 3.4.3 建筑状态机
```
          开始建造
             ↓
          建造中 (1分钟,可付费加速)
             ↓
          待激活
             ↓
        玩家点击激活
             ↓
          运行中 ←→ 暂停 (可切换)
             ↓
          损坏 (耐久度耗尽)
             ↓
        维修 → 运行中
```

**状态说明**:
- 建造中: 不可使用,显示进度条
- 待激活: 建造完成,等待玩家激活
- 运行中: 正常工作,产出资源
- 暂停: 不产出,不消耗耐久
- 损坏: 需要消耗材料维修

#### 3.4.4 建筑升级
- 升级条件: 建筑必须处于"运行中"状态
- 升级成本: 原建造成本 × 1.5
- 升级时间: 5分钟
- 升级效果:
  - Lv1 → Lv2: 产出+50%
  - Lv2 → Lv3: 产出+100%, 解锁特殊功能
  - 最高等级: Lv3

### 3.5 交易系统

#### 3.5.1 交易机制
**设计理念**: 以物易物,去中心化,基于地理位置的信任交易

**交易流程**:
1. 玩家A发布挂单:
   - 我提供: 罐头×30
   - 我需要: 药品×10
2. 玩家B在100米范围内看到挂单
3. 玩家B点击"接受交易"
4. 系统检查双方资源
5. 资源同时扣除和增加
6. 交易完成,挂单下架

**交易限制**:
- 必须在交易开放的领地100米内
- 必须有足够的背包空间
- 每天交易次数上限: 免费10次,会员无限
- 交易冷却: 同一玩家5分钟内只能交易一次

#### 3.5.2 交易界面
**挂单展示**:
```
[玩家头像] PlayerName  距离: 85m
━━━━━━━━━━━━━━━━━━━━━━━━━━━
提供: 🥫罐头 ×30
需要: 💊药品 ×10
━━━━━━━━━━━━━━━━━━━━━━━━━━━
         [接受交易]
```

**我的挂单管理**:
- 查看所有挂单
- 编辑挂单
- 取消挂单
- 交易历史

#### 3.5.3 交易信用系统(未来版本)
- 成功交易+1信用
- 取消交易-1信用
- 信用等级影响交易优先级
- 低信用玩家交易受限

### 4.2 生存体征系统

#### 4.2.1 核心生命值
- 初始值: 100
- 最大值: 100(可通过建筑提升至150)
- 死亡: 归零后角色死亡,复活需消耗资源
- 恢复方式:
  - 在庇护所休息: +10/小时
  - 使用急救包: +30
  - 使用抗生素: +50

#### 4.2.2 饱食度
- 初始值: 100%
- 衰减速度: -5%/小时
- 低于20%: 开始掉血(-5生命/小时)
- 高于80%: 探索速度+20%
- 恢复方式:
  - 罐头食品: +40%
  - 干粮: +20%
  - 水果: +15%

#### 4.2.3 水分
- 初始值: 100%
- 衰减速度: -10%/小时(比饱食度快)
- 低于20%: 开始掉血(-10生命/小时)
- 高于80%: 资源获取+20%
- 恢复方式:
  - 饮用水: +30%
  - 净化水: +50%

#### 4.2.4 状态效果
**正面BUFF**:
- 营养充足(饱食>80% 且 水分>80%): 全属性+20%
- 休息良好(在庇护所): 经验+10%
- 装备加成(高级装备): 各种加成

**负面DEBUFF**:
- 饥饿(饱食<20%): 移动速度-30%, 掉血
- 脱水(水分<20%): 视野范围-50%, 掉血
- 受伤(生命<30%): 无法建造,探索受限

## 8. 平衡性设计

### 8.1 数值平衡

#### 8.1.1 探索产出平衡
**设计目标**: 每小时探索可获得足够维持1小时生存的资源

计算:
- 每小时消耗: 食物5%, 水10%, 约需罐头×2, 水×3
- 每次探索平均产出: 罐头×24, 水×12
- 结论: 1次探索 = 12小时生存 ✓ 平衡

#### 8.1.2 建造成本平衡
**设计原则**: 建筑成本 = 预期探索次数 × 平均产出

示例: 篝火
- 成本: 石头×20, 木材×30
- 石头平均产出: 5个/次 → 需要4次探索
- 木材平均产出: 10个/次 → 需要3次探索
- 总计: 约4次探索可建造1个篝火 ✓ 合理

#### 8.1.3 升级成本曲线
```
Lv1→Lv2成本 = 原成本 × 1.5
Lv2→Lv3成本 = 原成本 × 2.5
```
确保后期玩家有持续目标,但不会过于困难。

---

### 8.2 付费平衡

#### 8.2.1 付费vs免费
**设计原则**: 付费玩家效率提升2-3倍,但不获得独占能力

对比:
| 能力 | 免费玩家 | 付费玩家(月卡) | 差距 |
|------|---------|---------------|------|
| 探索次数 | 10次/天 | 无限 | 效率×3 |
| 资源获取 | 100% | 150% | 效率×1.5 |
| 背包容量 | 100格 | 150格 | 空间×1.5 |
| 圈地能力 | ✓ | ✓ | 无差距 |
| 建造能力 | ✓ | ✓ | 无差距 |
| 交易能力 | ✓ | ✓ | 无差距 |

**结论**: 付费玩家更快达成目标,但免费玩家通过时间也能达成 ✓ 平衡

#### 8.2.2 付费转化率预估
- 首周付费率: 5-8%
- 长期付费率: 15-20%
- 目标ARPU: $2-5

## 15. 设计哲学

### 15.1 核心设计原则
1. **真实性**: 游戏机制贴近现实逻辑
2. **公平性**: 付费不影响核心竞争力
3. **社交性**: 鼓励玩家互动合作
4. **持续性**: 长期游玩价值
5. **创新性**: 独特的LBS+生存结合

### 15.2 玩家体验目标
- **成就感**: 通过圈地、建造、排名获得
- **探索欲**: 发现新POI、新区域的好奇心
- **社交感**: 与其他玩家交易、聊天、组队
- **进步感**: 等级提升、资源积累、能力增强
- **沉浸感**: 世界观、音效、视觉的整体体验
